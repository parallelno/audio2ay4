## ⚠️ CRITICAL FINDING (2026-06, human listening test) — THE SPECTRAL METRIC IS PERCEPTUALLY INVALID
- A human A/B listened to `samples/long/The_Last_Pixel.mp3` rendered by 4 methods (files in `samples/long/compare/`). Verdict by EAR:
  - **dummy** (deterministic heuristic): resembles the original theme, primitive but **most notes present**, noise channel does drums. Melodically the strongest overall.
  - **v7 warm-start**: **main theme recognizable**, weak, some notes missing/untuned, noise≈drums. **Best of the LEARNED models.** Still worse than the old **audio2ay3** project.
  - **v1 reward (A4 keeper)**: only drums recognizable, **main theme GONE**. Bad.
  - **v3 aug (A5)**: monotonic beeping + short noise bursts, theme GONE. Bad.
- Our `spec_dist` ranked these the **near-OPPOSITE** (v1 0.206 "best" → v7 0.209 → v3 0.242 "worst"). **Multi-scale STFT-magnitude distance does NOT measure musical quality.** A steady spectrally-averaged beep scores LOW spec_dist while containing NO melody.
- THEREFORE: **A4 (reward) and A5 (augment) are PERCEPTUAL REGRESSIONS** — they metric-gamed, destroying the melody the warm-start had. v1 is NOT the real keeper; **v7 warm-start is the best learned model by ear.**
- The **jitter penalty was part of the damage**: melody REQUIRES frequent note changes; rewarding low "stability" pushed toward monotonic output. "steadier=better" is WRONG for melodic content. (This invalidates the earlier "jitter is a helpful regularizer" conclusion — it was helpful only for the invalid metric.)
- ROOT CAUSE = exactly the INPUT DOMAIN note below: must reward **pitch/onset/chroma (right notes + rhythm)**, NOT spectral magnitude. Spectral-magnitude match has an irreducible floor AND optimizing it past a point regresses perception.
- NEXT (mandatory before any more reward training): build a **melody-aware EVAL metric** (chroma cosine / note-F1 via pitch tracking) so we measure what the ear hears; THEN a melody-aware reward term. Also investigate what **audio2ay3** did (it sounds better — likely explicit pitch detection / note tracking rather than learned spectral reward).

## INPUT DOMAIN (important constraint)
- Input is NOT real instruments. Input = SUNO-generated **synthetic chiptune-imitation** audio (AI music that mimics chiptune style, rendered as ordinary full-band/stereo/lossy audio).
- Consequence: narrow domain gap vs YM-rendered training audio (much closer than real instruments) → Plan A self-supervised reward fits well.
- Suno may exceed AY budget (>3 voices, drums, effects) → model must ARRANGE/reduce down to 3 tone + 1 noise + 1 env; perfect reproduction impossible.
- Reward/metric should weight musical structure (pitch/onset/chroma) over exact timbre (we translate Suno-timbre → AY-timbre). Spectral-magnitude match has an irreducible floor.
- Augmentation in data/ should mimic Suno coloration (stereo→mono, reverb, EQ, lossy MP3 codec, limiting) to bridge YM-render → Suno gap.
- Real-task eval set = a few SUNO chiptune clips (NOT real instruments). Two ML plans in `design/`: Plan A = RL reverse player, Plan B = conditional diffusion. Milestone 0 skeleton implemented.

## Plan A — A3 chip/diff differentiable emulator (DDSP) — DONE + validated vs oracle
- WHY (not reuse audio2ay3): reward training (A4) is analysis-by-synthesis — push gradient from audio-recon loss back into the policy. `audio2ay3` renders regs→audio FORWARD only (integer counters, LFSR, branches) → no ∂audio/∂regs, can't backprop. Build a differentiable DDSP twin so Regime-1 backprop works; `audio2ay3` stays the GROUND TRUTH (final .ym, eval) + the ORACLE the twin is validated against. Twin = approximate-but-differentiable; trades bit-exactness for gradients (matches oracle spectrally, which is all the gradient needs).
- Files: `src/audio2ay4/chip/diff/__init__.py` + `emulator.py` (torch-only, NOT exported from `chip/__init__.py` so the chip package stays torch-free; imported lazily during training, like models/policy/network.py). - `DiffAyEmulator(nn.Module)` mirrors `Ay3Emulator` conventions exactly: render at `render_sr*oversample` (44100*2) then box-average down; per-frame→per-sample expansion via `round(arange*spf)` boundaries (matches oracle's `int((f+1)*spf+0.5)`); active-low tone/noise gating (gate off ⇒ pass-through 1.0); mix `sum_c(amp_c * tone_c * noise_c)/3`. Registers map via datasheet: f_tone=clk/(16·TP), f_noise=clk/(16·NP), f_env=clk/(256·EP).
  * **Tone**: additive band-limited unipolar square (mean 0.5), `0.5 + (2/π)Σ_{k odd, k·f<Nyq} sin(kφ)/k`, `max_partials=32`. Phase = cumsum(2π·f/sr) in **float64** then `remainder(·,2π)` (float32 sin loses precision at large cumulative phase). Differentiable wrt TP.
  * **Noise**: replicates the oracle's 17-bit LFSR (taps 0,3) into a precomputed `{0,1}` buffer (`_lfsr_sequence`, len 2^20), sample-held by `floor(cumsum(f_noise/sr))` index. Matches oracle spectrum; gradient wrt NP ~0 (floor) — acceptable (design clamps noise expressiveness anyway).
  * **Envelope**: resettable 16-step ramp/triangle/hold for all 16 R13 shapes via closed-form `pos`=cumsum(env-steps) with reset at retrigger (`csum - cummax(offset@reset)` trick), `q=pos%16`, decomposed by shape bits cont/att/alt/hold; DAC table **linearly interpolated** at continuous level so gradient flows wrt EP. Held level = `(att xor alt)?15:0`.
  * **DAC**: `AY_DAC` mirrored as a module constant (NOT imported from audio2ay3 — keeps twin self-contained; test asserts it equals the oracle table). `_dac_interp` for sub-level gradients.
- `DiffControls` dataclass (continuous per-frame tensors: tone_period(n,3)/noise_period(n)/env_period(n)/tone_gate/noise_gate(n,3 in[0,1])/level(n,3 in[0,15])/use_env(n,3)/env_shape(n,long)/env_retrig(n)) = the smooth space the policy emits (integer register packing BYPASSED for the grad path). `unpack_regs(regs)` (numpy bit-unpack, non-diff) builds DiffControls from a uint8 stream — used to feed the twin the SAME input as the oracle for validation, and to bootstrap from parsed corpus regs. `render(controls,…)` for training, `render_regs(regs,…)` convenience for oracle comparison.
